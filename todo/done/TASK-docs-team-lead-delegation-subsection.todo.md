# TASK-docs-team-lead-delegation-subsection: Подраздел «Делегирование» в стиле работы Тимлида

## 1. Concept and Goal (Концепция и Цель)
### Story (Job Story)
Когда пользователь даёт Тимлиду задачу, Тимлид должен предложить план и делегировать исполнение, а не выполнять сам. Сейчас в роли `team_lead.ru.md` этот принцип заявлен («Тимлид не пишет код сам»), но не формализован: нет чёткого правила о предложении задачи пользователю перед действием.

### Goal (Цель по SMART)
Добавить подраздел «Делегирование» в конец раздела «Стиль работы» файла `docs/agents/roles/team/team_lead.ru.md`. Подраздел содержит два явных правила делегирования.

## 2. Context and Scope (Контекст и Границы)
* **Где делаем:**
  * `docs/agents/roles/team/team_lead.ru.md` — добавление подраздела в «Стиль работы»
* **Текущее поведение:** В описании роли есть фраза «Тимлид не пишет код сам», но нет правила о предложении задачи пользователю и о запрете самостоятельного выполнения.
* **Границы (Out of Scope):**
  * Другие роли — не трогаем
  * AGENTS.md — не трогаем (общие правила уже там)
  * Шаблоны задач и скиллы — не трогаем
