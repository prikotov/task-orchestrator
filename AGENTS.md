# AGENTS.md — prikotov/task-orchestrator

Обязательные правила для AI-агента в проекте task-orchestrator.

---

# Роль

* Перед выполнением запроса пользователя загрузи роль из:
    - [`Бэкендер`](vendor/prikotov/coding-standard/doc/AGENTS.md) — конвенции по умолчанию для PHP/DDD разработки.

---

# Рефлексия

* Перед выполнением запроса:
  - оцени `сложность запроса` от 0 до 10;
  - оцени `уровень контекста` от 0 до 10;
  - оцени `риск ошибки` от 0 до 10.
* Запрос считается **проблемным**, если `сложность` >= 7 или `контекст` <= 4 или `риск` >= 7.
* Для проблемного запроса — укажи классификацию, перечисли допущения, предложи план и дождись подтверждения.

---

# Язык

* Общайся с пользователем только на русском языке.
* Технические сущности называй на английском.

---

# Архитектура проекта

* Symfony Bundle, PHP 8.4, DDD.
* **Структура:**
  - `src/Domain/` — сущности, VO, интерфейсы, доменные сервисы.
  - `src/Application/` — use case, DTO, мапперы.
  - `src/Infrastructure/` — репозитории, внешние интеграции.
  - `tests/Unit/` — unit-тесты.
  - `tests/Integration/` — integration-тесты.

---

# Работа с кодом

* Прямые правки в `master` запрещены — работай в ветке `task/<short-description>`.
* Соблюдай модульность и слоистую архитектуру.
* Любое изменение сопровождай тестами.
* Коммиты — Conventional Commits.
* Документируй публичные классы и интерфейсы.
