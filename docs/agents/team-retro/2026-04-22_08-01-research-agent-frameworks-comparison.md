# Ретроспектива: EPIC-research-agent-frameworks-comparison

**Дата:** 2026-04-22
**Эпик:** `EPIC-research-agent-frameworks-comparison` (11 задач, 11 PR → 1 финальный PR #51)
**Что делали:** Исследование 13 AI-agent фреймворков и оркестраторов. Сравнение с task-orchestrator. 10 индивидуальных comparison-отчётов + сводная таблица с рекомендациями.

## Оценка ролей

### Тимлид (Алекс) (`docs/agents/roles/team/team_lead.ru.md`)
- **Плюсы:**
  - Эпик грамотно структурирован: 10 параллельных задач (Этап 1) + 1 финальная (Этап 2). Все задачи были независимы, что позволило выполнять их последовательно без блокировок.
  - Все задачи формализованы до старта: YAML front matter, критерии приёмки, DoD,MoSCoW — сабагенты получали исчерпывающий контекст.
  - Делегирование через `run-pi-subagent` работало стабильно: ни одна задача не потребовала ручного вмешательства Тимлида в content.
  - Сабагенты корректно обновляли summary-таблицу инкрементально (каждый добавлял свою строку + рекомендации + тренды).
- **Минусы:**
  - Задача TASK-research-crewai-langgraph-autogen (C4, 3 фреймворка в одной задаче) оказалась слишком объёмной для одного сабагента — первый запуск timed out (2700s), потребовался повтор.
  - У двух задач после завершения сабагентом остался некорректный статус: `claude-code` — `in_progress` вместо `done`, `summary` — пустой `pr:`. Тимлид фиксил это вручную при подготовке ретро.
- **Предложения по улучшению:**
  - Разбивать задачи C4 с 3+ фреймворками на отдельные задачи по одному фреймворку (C3 каждая).
  - Добавить в инструкции сабагенту явное требование: «Убедись что `status: done` и `pr: <url>` заполнены перед commit».

### Технический писатель (Гермиона) (`docs/agents/roles/team/technical_writer.ru.md`)
- **Плюсы:**
  - Все 10 comparison-документов выполнены по единому формату (Обзор → Сравнительная таблица → Что полезно → Что НЕ берём → Сводка рекомендаций → Источники). Единообразие отличное.
  - Сабагенты корректно обновляли summary-таблицу при каждом запуске — не перетирали чужие строки, а добавляли свои + рекомендации + тренды.
  - Финальная задача (summary) полноценно проанализировала все 10 отчётов и выдала кластеризованные рекомендации с приоритетами.
  - Каждый сабагент самостоятельно перемещал задачу в `done/` и обновлял эпик — Тимлиду оставалось только merge PR.
- **Минусы:**
  - Не все сабагенты заполняли `pr:` и `status:` корректно — 2 из 11 задач имели неполные метаданные после завершения.
  - Финальный сабагент (summary) не обновил чекбоксы Must Have / Should Have в эпике — Тимлид фиксил вручную.
- **Предложения по улучшению:**
  - Добавить в промпт сабагента чеклист: «Перед commit проверь: (1) status: done, (2) pr: заполнен, (3) задача перемещена в done/, (4) эпик обновлён, (5) все чекбоксы в эпике отмечены».

## Что прошло хорошо

- **Стабильный pipeline делегирования.** 11 из 11 задач выполнены через `watch-subagent.sh` — ни одна не потребовала ручного написания контента Тимлидом. Процесс «создать ветку → обновить задачу → push → draft PR → запустить сабагента → merge» отработал как конвейер.
- **Инкрементальное заполнение summary.** Вместо одного большого сабагента для всей таблицы — каждый сабагент добавлял свою строку. Это снизило риск потери данных и позволило накапливать тренды по мере исследования.
- **Единый формат отчётов.** Все 10 comparison-документов следуют одной структуре. Сводная таблица имеет 13/13 заполненных строк без пропусков.
- **Корректная работа с git.** Сабагенты не ломали чужие ветки, корректно branching от epic-ветки, не трогали чужие файлы в summary.

## Проблемы

- **Таймаут сабагента на C4 задаче (TASK-research-crewai-langgraph-autogen, PR #42).** Задача требовала исследовать 3 фреймворка (CrewAI, LangGraph, AutoGen) — это C4 complexity. Первый запуск `watch-subagent.sh` с `-s 2700` завершился по таймауту, но сабагент успел закоммитить результат. Это не блокировало задачу, но потребовало ручной проверки completeness.
- **Неполные метаданные задач после завершения сабагента.** В 2 из 11 задач сабагент не обновил `status:` (остался `in_progress`) или `pr:` (остался пустым). Тимлид обнаружил это только при подготовке ретро — значит, в общем flow нет валидации метаданных перед merge.
- **Чекбоксы эпика не обновлены финальным сабагентом.** Сабагент summary отметил задачу `[x]` и обновил DoD, но не тронул Must Have / Should Have в секции 3 эпика. Тимлид фиксил через `sed`.
