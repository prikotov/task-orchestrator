## Что прошло хорошо

- **Стабильный pipeline делегирования.** 11 из 11 задач выполнены через `watch-subagent.sh` — ни одна не потребовала ручного написания контента Тимлидом. Процесс «создать ветку → обновить задачу → push → draft PR → запустить сабагента → merge» отработал как конвейер.
- **Инкрементальное заполнение summary.** Вместо одного большого сабагента для всей таблицы — каждый сабагент добавлял свою строку. Это снизило риск потери данных и позволило накапливать тренды по мере исследования.
- **Единый формат отчётов.** Все 10 comparison-документов следуют одной структуре. Сводная таблица имеет 13/13 заполненных строк без пропусков.
- **Корректная работа с git.** Сабагенты не ломали чужие ветки, корректно branching от epic-ветки, не трогали чужие файлы в summary.
- **Постфактум ревью сработал.** 5 критических и 15+ значимых ошибок найдено во втором прогоне. Верификация по исходникам (git clone) и документации (curl) — ключевой фактор качества.
- **Распределение ролей по типу продукта.** Пуаро → open-source (верификация по коду), Локи → проприетарные (критика по документации) — каждый ревьювер работал в своей сильной зоне.

## Проблемы

- **Таймаут сабагента на C4 задаче (TASK-research-crewai-langgraph-autogen, PR #42).** Задача требовала исследовать 3 фреймворка (CrewAI, LangGraph, AutoGen) — это C4 complexity. Первый запуск `watch-subagent.sh` с `-s 2700` завершился по таймауту, но сабагент успел закоммитить результат. Это не блокировало задачу, но потребовало ручной проверки completeness.
- **Неполные метаданные задач после завершения сабагента.** В 2 из 11 задач сабагент не обновил `status:` (остался `in_progress`) или `pr:` (остался пустым). Тимлид обнаружил это только при подготовке ретро — значит, в общем flow нет валидации метаданных перед merge.
- **Чекбоксы эпика не обновлены финальным сабагентом.** Сабагент summary отметил задачу `[x]` и обновил DoD, но не тронул Must Have / Should Have в секции 3 эпика. Тимлид фиксил через `sed`.
- **PR #51 смержен без ревью.** Исследования приняли как docs-only и пропустили проверки. Итог — 5 критических фактических ошибок попали в `main`. Docs-only ≠ не нужны проверки: исследовательские отчёты — основа для архитектурных решений.
- **5 из 11 сабагентов-ревьюверов ушли в таймаут** (900 сек). Причина: клонирование репозитория + чтение десятков файлов + генерация правок в одном прогоне.
- **Все 13 вердиктов одинаковые — 🟡 «заимствовать паттерны».** Либо закономерность (ни один не подходит как dependency для PHP-проекта), либо слабая дифференциация. Стоило дать хотя бы один 🔴 или 🟢.
- **Прямой коммит в main** — Тимлид закоммитил обновление эпика напрямую в `main`, нарушив правило «только через PR».
- **Сводная таблица использует нумерацию** `*(в отчёте №3)*` вместо прямых markdown-ссылок на файлы.
