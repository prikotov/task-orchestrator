# Отчёт агента

Сохранение результата работы агента в файл. Каждый отчёт — self-contained markdown, который можно прочитать без контекста запуска.

## Когда использовать

- Агент завершил ревью, аудит или анализ (архитектурное ревью, код-ревью, тест-репорт)
- Нужно сохранить результат для истории, прослеживаемости или передачи другому агенту

## Как использовать

### **Шаг 1:** Создание файла отчёта

Файл: `docs/agents/reports/<role-slug>/<YYYY-MM-DD_HH-MM>_<short-description>.md`

Параметры пути файла:

| Параметр          | Описание                                   | Пример                    |
|-------------------|--------------------------------------------|---------------------------|
| role-slug         | Идентификатор роли на англ., kebab-case    | code-reviewer-backend     |
| YYYY-MM-DD        | Дата запуска сабагента                     | 2026-04-19                |
| HH-MM             | Местное время запуска                      | 14-30                     |
| short-description | Краткое описание отчёта на англ., kebab-case | architecture-review     |

Примеры:

```bash
    docs/agents/reports/system-architect/2026-04-19_14-30_architecture-review.md
    docs/agents/reports/code-reviewer-backend/2026-04-20_09-15_pr-27-review.md
    docs/agents/reports/qa-backend/2026-04-20_16-45_task-xyz-test-report.md
```

### **Шаг 2:** Заполнение шапки отчёта

Каждый отчёт начинается с обязательной шапки:

```markdown
# <Заголовок отчёта>

**Роль:** <Имя роли>
**Дата:** YYYY-MM-DD
**Объект:** <что анализировалось — файлы, модули, PR>
**Задача:** <ссылка на задачу или причина запуска>

---
```

### **Шаг 3:** Написание тела отчёта и сохранение

1. После шапки напиши основное содержимое отчёта: находки, замечания, рекомендации — в зависимости от типа отчёта.
2. Убедись, что отчёт self-contained — читается без контекста запуска.

### **Шаг 4:** Вывод резюме пользователю

Выведи краткое резюме и ссылку на файл, пример:

```
📄 Отчёт: <role-slug>/<filename>.md

Кратко:
- 🔴 <критические находки>
- 🟡 <важные замечания>

Полный отчёт: docs/agents/reports/<role-slug>/<filename>.md
```

## Результат

Файл: `docs/agents/reports/<role-slug>/<YYYY-MM-DD_HH-MM>_<short-description>.md`

Чеклист самопроверки:

- [ ] Файл создан и именован по конвенции: `YYYY-MM-DD_HH-MM_short-description.md` — проверь перед сохранением, что имя файла соответствует шаблону
- [ ] Шапка заполнена: роль, дата, объект, задача — без шапки отчёт теряет контекст, другой агент не поймёт, кто и зачем его написал
- [ ] Отчёт self-contained — читается без контекста запуска; если отчёт ссылается на файлы, укажи полные пути
- [ ] Пользователь получил краткое резюме со ссылкой на файл — без этого отчёт «пропадёт» в stdout и будет потерян
