# Ретроспектива

Самоулучшение после завершения эпика или задачи: зафиксировать сильные практики, найти слабые, чтобы следующая итерация прошла быстрее и чище.

## Когда использовать

- Эпик завершён (финальный PR создан или влит)
- После серии задач с повторяющимися проблемами
- Пользователь явно попросил провести ретро
- После горячего исправления (hotfix), если процесс выявил системные проблемы

## Как использовать

### **Шаг 1:** Сбор контекста

1. Прочитай файл эпика (`todo/done/EPIC-*.todo.md`).
2. Прочитай все выполненные задачи эпика.
3. Прочитай существующие ретроспективы в `docs/agents/team-retro/` для понимания повторяющихся проблем.

### **Шаг 2:** Анализ ролей

Для каждой задействованной роли:

1. Оцени поведение роли по трём критериям:
   - **Плюсы:** что сработало хорошо, где роль проявила себя.
   - **Минусы:** что не сработало, где не хватило инструкций или компетенций.
   - **Предложения по улучшению:** конкретные изменения в файл роли или процессе.

2. Фиксируй конкретные факты. Плохо: «роль справилась». Хорошо: «Тестировщик обнаружил 3 бага в валидации VO, не покрытых unit-тестами (TASK-foo)».

### **Шаг 3:** Анализ процесса

Проанализируй по направлениям:

- **Процессы:** что было неочевидно или нарушалось; таймауты, координация.
- **Конвенции:** что было неочевидно или нарушалось.
- **Инструменты:** проблемы сабагентов, git, CI.
- **Координация:** ошибки при передаче задач между ролями.
- **Повторяющиеся:** проблемы, которые уже были в прошлых ретро.

Для каждого направления:
- Если проблем нет — укажи «Без замечаний».
- Если есть — опиши конкретно: файл, задача, симптом.

### **Шаг 4:** Проверка на повторы

1. Прочитай все прошлые ретроспективы из `docs/agents/team-retro/`.
2. Сравни найденные проблемы с историческими.
3. Если проблема повторяется — отметь это: «⚠️ Повтор из ретро `<имя файла>`».
4. Повторяющаяся проблема — приоритетнее для исправления.

### **Шаг 5:** Формирование предложений

1. На основе анализа из **Шагов 2–4** сформируй 1–3 конкретных предложений по самым критичным проблемам.
2. Каждое предложение содержит:
   - Конкретное действие.
   - Файл или процесс, которые нужно изменить.
   - Статус: внедрено / TODO / требует обсуждения.

3. Приоритизация предложений:
   - 🔴 Повторяющиеся проблемы из прошлых ретро.
   - 🟡 Нарушения конвенций и процессов.
   - 🟢 Мелкие улучшения и оптимизации.

### **Шаг 6:** Запись ретроспективы

Создай файл по шаблону [`retrospective-template.md`](assets/retrospective-template.md).

```bash
    docs/agents/team-retro/<YYYY-MM-DD_HH-MM>-<slug>.md
```

Параметры имени файла:

| Параметр | Описание                                          | Пример               |
|----------|---------------------------------------------------|----------------------|
| YYYY     | Год                                               | 2026                 |
| MM       | Месяц                                             | 04                   |
| DD       | День                                              | 19                   |
| HH-MM    | Местное время                                     | 14-30                |
| slug     | Короткий идентификатор эпика на англ., kebab-case | task-orchestrator-v2 |

Примеры:

```bash
    docs/agents/team-retro/2026-04-19_14-30-00-docs-architecture-sync.md
    docs/agents/team-retro/2026-05-02_09-15-30-chain-builder-epic.md
```

Содержимое файла заполни по структуре шаблона:

- **Заголовок + метадата** *(обязательна)* — дата, эпик, количество задач/PR, что делали.
- **Оценка ролей** *(обязательна)* — плюсы, минусы, предложения для каждой роли.
- **Что прошло хорошо** *(обязательна)* — конкретные факты, которые стоит повторять.
- **Проблемы** *(обязательна)* — описания с привязкой к задачам/файлам.
- **Предложения для улучшения** *(обязательна)* — конкретные шаги с чекбоксами и указанием файлов.
- **Важные комментарии** *(опциональна)* — нарушения конвенций, неочевидные правила.

### **Шаг 7:** Отчёт пользователю

Выведи краткое резюме:

```
📋 Ретроспектива: <slug>

✅ Что прошло хорошо:
- <1–3 пункта>

⚠️ Проблемы:
- <1–3 пункта>

🔧 Предложения:
- <1–3 пункта>

Файл: docs/agents/team-retro/<filename>.md
```

Если есть предложения, требующие действий пользователя (статус «требует обсуждения») — явно укажи их.
