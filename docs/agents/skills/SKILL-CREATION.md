# Создание SKILL

Инструкция по созданию навыков для AI-агентов.

SKILL (навык) — это переносимый модуль (инструкции + скрипты), позволяющий агенту автономно выполнять конкретные задачи, такие как работа с API, анализ файлов или другие специфические действия. Это стандарт для структурирования опыта, который агент загружает по мере необходимости.

## Структура

```
skill-name/
├── SKILL.md          # Обязательный — основная инструкция
├── scripts/          # Опционально — исполняемые скрипты
├── references/       # Опционально — справочные материалы
└── assets/           # Опционально — шаблоны, примеры файлов
```

## SKILL.md

### YAML Frontmatter (обязательно)

```yaml
---
name: skill-name
description: Что делает skill (кратко).
---
```

**Требования:**
- `name`: kebab-case, только строчные, без пробелов
- `description`: краткое описание функции skill (до 1024 символов)
- Без XML-тегов (`< >`)

### Тело SKILL.md

`````
# Skill Name

## Когда использовать

- Ситуация 1
- Ситуация 2

## Как использовать

**Шаг 1:** Название шага

```bash
    command [опции] [аргументы]
```

Опции:

| Опция | Описание | Значения | По умолчанию |
|-------|----------|----------|--------------|
| --param | Описание | val1, val2 | val1 |

Примеры:

```bash
    command --param=val1
    command --param=val2 arg
```

**Шаг 2:** Название шага

    another-command [опции]

## Результат

Файл: `path/to/output.csv`

Поля:

| Поле | Описание |
|------|----------|
| field | Описание поля |
`````

## Разделы

| Раздел                 | Описание                                     |
|------------------------|----------------------------------------------|
| **Когда использовать** | Контекст применения                          |
| **Как использовать**   | Шаги с командами, опциями, примерами         |
| **Результат**          | Описание того что и где получим в результате |

## Шаги

Каждый шаг содержит:

1. **Инструкция шага** — описание того что нужно сделать агенту
2. **Команда** (опционально) — синтаксис вызова 
3. **Опции** (опционально) — таблица с параметрами
4. **Результат шага** (опционально) — таблица с параметрами
5. **Примеры** (опционально) — конкретные вызовы

`````
**Шаг 1:** Название
```
    command [опции]
```
Опции:

| Опция | Описание | Значения | По умолчанию |
|-------|----------|----------|--------------|
| --sort | Поле сортировки | visits, date | visits |
| --limit | Лимит записей | число | все |

Примеры:

```bash
    command --sort=date --limit=10
    command --limit=20
```
`````

## Принципы

1. **description краткий** — только что делает
2. **Структура** — когда → как → результат
3. **Шаги** — инструкция + команда + опции + примеры
4. **Результат** — описание + что и где получим

## Пример

```yaml
---
name: yandex-metrika-search
description: Поисковые фразы из Яндекс.Метрики для SEO-анализа
---
```

`````
# Yandex Metrika Search

## Когда использовать

- Анализ поискового трафика
- Поиск популярных запросов
- Определение фраз с высокими отказами

## Как использовать

```bash
    php metrika.php [опции] [дата_от] [дата_до]
```

Параметры:

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| дата_от | Начало периода (YYYY-MM-DD) | 30 дней назад |
| дата_до | Конец периода (YYYY-MM-DD) | сегодня |

Опции:

| Опция | Сокращение | Описание | Значения | По умолчанию |
|-------|------------|----------|----------|--------------|
| --site | | Имя сайта | имя сайта | default_counter |
| --sort | -s | Поле сортировки | visits, bounce_rate, page_depth | visits |
| --order | -o | Направление | asc, desc | desc |
| --limit | -l | Лимит записей | число | все |

Примеры:

```bash
    # Топ-10 фраз
    php metrika.php --limit 10

    # Фразы с высоким отказом
    php metrika.php -s bounce_rate -l 20

    # За период
    php metrika.php -l 10 2025-01-01 2025-01-31
```

## Результат

Файл: `yandex_metrika_reports/YYYY-MM-DD/yandex_metrika_phrases.csv`

Поля:

| Поле | Описание |
|------|----------|
| phrase | Поисковая фраза |
| visits | Визиты |
| visitors | Посетители |
| bounce_rate | Процент отказов |
| page_depth | Глубина просмотра |
| avg_duration | Среднее время на сайте (сек) |
`````
