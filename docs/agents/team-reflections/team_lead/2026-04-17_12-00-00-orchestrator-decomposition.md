# Рефлексия: Декомпозиция модуля Orchestrator

**Дата:** 2026-04-17
**Эпик:** `EPIC-arch-orchestrator-module-decomposition` (4 задачи, 4 PR)
**Что делали:** Декомпозиция модуля Orchestrator (110 файлов) на AgentRunner (~18 файлов) и Orchestrator (~103 файла) с Port/Adapter изоляцией.

## Что прошло хорошо

- Алгоритм Шаг 0→8 — чёткий конвейер, не было моментов «а что сейчас делать?»
- Маппинг «тип задачи → роль» — таблица сработала без сбоев
- Шаблоны промптов и bash-команд — всё работало из коробки
- Цикл реализация → ревью → доработка — отработал чисто, максимум 1 итерация доработки
- Конкретный пример в конце team_lead.ru.md — «живой» образец, который можно копировать

## Проблемы

1. **Таймауты сабагентов (2 из 4 задач).** TASK-2 (C3, 600 сек) и TASK-3 (C4, 900 сек) — массовая миграция namespace. Сабагент обновил `src/`, но не успел обновить `tests/`.
   → **решение:** добавлена эмпирическая таблица таймаутов по сложности (C1–C2: 300, C3: 600, C4: 900, C5+: 1200 сек), добавлен протокол обработки таймаутов в Шаг 4.

2. **Сабагенты не делают промежуточные коммиты.** При таймауте — незакоммиченный «мусор» в рабочем каталоге.
   → **решение:** добавлен пункт о промежуточных коммитах в шаблон инструкций для сабагента.

3. **`gh pr create --body-file -` не работает.** В шаблоне указан `--body-file -` с heredoc — ошибка `must provide --title and --body`.
   → **решение:** заменено на `--title "..." --body "..."`.

4. **Подветки задач созданы от `master` вместо ветки эпика.** TASK-arch-agent-runner-application-layer — сабагент не нашёл файлов для рефакторинга, т.к. код AgentRunner был только на ветке эпика.
   → **решение:** добавлено критическое предупреждение в модель работы + `git pull` в шаблон Шага 2.

5. **Сабагент не закоммитил изменения после завершения.** Доработка по CR — изменения в рабочем каталоге, но не закоммичены.
   → **решение:** добавлен сценарий в таблицу обработки ошибок — тимлид делает коммит сам.
