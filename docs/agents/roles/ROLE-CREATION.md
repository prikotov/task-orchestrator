# Создание ROLE

Инструкция по созданию ролей для AI-агентов.

ROLE (роль) — это описание персонажа-агента с характером, компетенциями и инструкциями поведения. Роль загружается агентом при старте задачи и определяет его подход к работе.

## Структура

```
roles/
└── team/
    ├── full_role_name.ru.md    # Файл роли (локаль в суффиксе)
    └── full_role_name.en.md    # Опционально — другая локаль
```

## Именование файлов

Файл роли: `{full_role_name}.{locale}.md`

- `full_role_name` — уникальный идентификатор роли в формате `snake_case`: `{role}_{name}`
- Совпадает с именем файла **без** расширения локали и `.md`

Примеры:

| Файл | full_role_name |
|---|---|
| `team_lead_alex.ru.md` | `team_lead_alex` |
| `system_architect_gandalf.ru.md` | `system_architect_gandalf` |
| `backend_developer_tony.ru.md` | `backend_developer_tony` |

## Front Matter (YAML)

### Поля front matter

Рекомендуется прописывать все поля. Порядок — как в таблице:

```yaml
---
full_role_name: <role_type>_<name>
role: <role_type>
name: "<Имя на языке файла>"
title: "<Роль + Имя на языке файла>"
personality:
  jung: ["<archetype>"]
  disc: "D<n> I<n> S<n> C<n>"
  belbin: ["<role>", "<role>"]
  adizes: ["<role>", "<role>"]
  big_five: "O<n> C<n> E<n> A<n> N<n>"
skills: ...
description: "Краткое описание ответственности роли."
---
```

| Поле | Формат | Описание | Пример |
|---|---|---|---|
| `full_role_name` | `snake_case`, латиница | Идентификатор = `role` + `_` + `name`, совпадает с именем файла | `team_lead_alex` |
| `role` | `snake_case`, латиница | Тип роли (без имени) | `team_lead`, `backend_developer` |
| `name` | Строка, язык файла | Имя персонажа | `"Алекс"`, `"Alex"` |
| `title` | Строка, язык файла | Полное название роли: роль + имя | `"Тимлид Алекс"`, `"Team Lead Alex"` |
| `personality` | YAML-объект | Психотип персонажа (см. «Поле personality» ниже) | |
| `skills` | YAML-массив | Список skill'ей, которые роль может использовать (ссылки на `docs/agents/skills/`) | |
| `description` | Строка | Краткое описание ответственности | `"Оркестратор эпиков."` |

### Поле personality

Описывает психотип персонажа через 5 моделей. Все подсекции опциональны, но рекомендуется заполнять все.

```yaml
personality:
  jung: ["<archetype>"]
  disc: "D<n> I<n> S<n> C<n>"
  belbin: ["<role>", "<role>"]
  adizes: ["<role>", "<role>"]
  big_five: "O<n> C<n> E<n> A<n> N<n>"
```

#### jung — Юнгианские архетипы

Тип личности по Карлу Юнгу. Указывается как массив (обычно один доминирующий архетип).

| Значение | Русский | Описание |
|---|---|---|
| `Caregiver` | Заботливый | Защита и помощь другим. Поддерживающий стиль. |
| `Creator` | Творец | Создание нового, вдохновляющий стиль. |
| `Everyman` | Славный малый | Принадлежность, демократичность, эмпатия. |
| `Explorer` | Искатель | Любознательность, независимость, R&D. |
| `Hero` | Герой | Уверенность, действие, тушение пожаров. |
| `Innocent` | Простодушный | Оптимизм, честность, happy path. |
| `Jester` | Шут | Игривость, ирония, нестандартные решения. |
| `Lover` | Любовник | Эстетика, страсть, UX/UI. |
| `Magician` | Маг | Визионерство, трансформация, автоматизация. |
| `Outlaw` (Rebel) | Бунтарь | Провокация, разрушение старого, рефакторинг. |
| `Ruler` | Правитель | Контроль, лидерство, распределение ресурсов. |
| `Sage` | Мудрец | Аналитика, объективность, поиск истины. |

📚 Справочник: [jung/](references/jung/)

#### disc — модель DISC

Поведенческий профиль. Формат: `D<n> I<n> S<n> C<n>`, где каждое значение от 0 до 10.

| Измерение | Расшифровка | Фокус |
|---|---|---|
| `D` — Dominance | Доминирование | Решение проблем, результат |
| `I` — Influence | Влияние | Люди, контакты, коммуникация |
| `S` — Steadiness | Стабильность | Темп, согласованность, процессы |
| `C` — Conscientiousness | Добросовестность | Процедуры, стандарты, качество |

📚 Справочник: [disc/](references/disc/)

#### belbin — роли Белбина

Командные роли. Указывается как массив (обычно 1–2 доминирующие роли).

| Значение | Русский | Описание |
|---|---|---|
| `Coordinator` | Координатор | Делегирование, уточнение целей, финальные решения. |
| `Specialist` | Эксперт | Узкоспециализированные знания, технические задачи. |
| `Completer Finisher` | Финишер | Внимание к деталям, поиск ошибок, полировка. |
| `Plant` | Генератор идей | Креативность, нестандартное мышление, брейншторминг. |
| `Implementer` | Исполнитель | Дисциплина, превращение идей в действия. |
| `Shaper` | Мотиватор | Драйв, преодоление препятствий, фокус на MVP. |
| `Resource Investigator` | Исследователь ресурсов | Поиск возможностей вне команды, анализ инструментов. |
| `Monitor Evaluator` | Стратег | Оценка рисков, стратегическое мышление, скептицизм. |
| `Teamworker` | Душа команды | Кооперация, дипломатия, сглаживание конфликтов. |

📚 Справочник: [belbin/](references/belbin/)

#### adizes — модель Адизеса

Управленческие роли. Указывается как массив (1–2 доминирующие роли).

| Значение | Буква | Описание |
|---|---|---|
| `Producer` | P | Результат, закрытые тикеты, кодинг. |
| `Administrator` | A | Процессы, стандарты, CI/CD, линтеры. |
| `Entrepreneur` | E | Идеи, видение, рефакторинг, новые технологии. |
| `Integrator` | I | Команда, согласие, связность модулей. |

📚 Справочник: [adizes/](references/adizes/)

#### big_five — Большая пятёрка (OCEAN)

Личностные черты. Формат: `O<n> C<n> E<n> A<n> N<n>`, где каждое значение от 0 до 10.

| Буква | Расшифровка | Высокое значение |
|---|---|---|
| `O` — Openness | Открытость опыту | Изобретательный, любопытный |
| `C` — Conscientiousness | Сознательность | Организованный, педантичный |
| `E` — Extraversion | Экстраверсия | Общительный, энергичный |
| `A` — Agreeableness | Доброжелательность | Дружелюбный, кооперативный |
| `N` — Neuroticism | Нейротизм | Чувствительный, тревожный |

📚 Справочник: [big5/](references/big5/)

## Тело файла (Markdown)

После front matter — полное описание роли на языке файла. Рекомендуемая структура:

```markdown
# <Title> (`<Роль>`)

**Цель:** <Одно предложение — зачем эта роль существует>

## Когда использовать
- <Ситуация 1>
- <Ситуация 2>

## Правила поведения
1. <Правило 1>
2. <Правило 2>

## Примеры
<Примеры задач и результатов>
```

## Принципы

1. **`full_role_name` = имя файла без `.{locale}.md`** — это уникальный идентификатор роли
2. **`title` на языке файла** — для `.ru.md` на русском, для `.en.md` на английском
3. **`full_role_name` всегда латиница** — совпадает с `role` + `_` + транслитерация `name`
4. **Одна роль — один файл на локаль** — `.ru.md`, `.en.md` и т.д.
5. **`role` без имени** — только тип (например `backend_developer`, не `backend_developer_tony`)

## Чеклист

- [ ] Файл в `docs/agents/roles/team/` (или другом подкаталоге)
- [ ] Имя файла: `{full_role_name}.{locale}.md`
- [ ] YAML front matter обрамлён `---`
- [ ] `role`: snake_case, латиница, без имени персонажа
- [ ] `name`: имя персонажа на языке файла
- [ ] `title`: роль + имя на языке файла, поле идёт после `name`
- [ ] `full_role_name`: совпадает с именем файла без `.{locale}.md`
- [ ] `description`: краткое описание ответственности
- [ ] `personality`: заполнены все 5 подсекций (jung, disc, belbin, adizes, big_five)
- [ ] Тело файла начинается с `# <Title>`
