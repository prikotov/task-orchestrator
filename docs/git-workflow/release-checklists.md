# Чеклисты релиза, срочного исправления и выкладки

Короткие рабочие чеклисты для подготовки релиза, срочного исправления и выкладки.

## Чеклист подготовки релиза

- Состав релиза согласован, лишние изменения остаются в `master`.
- От актуального `master` создана одна активная `release/x.y`.
- Все PR для стабилизации целятся в `release/x.y`.
- Для релиза выполнены `make check` и `make tests-e2e`.
- `CHANGELOG.md` и описание релиза подготовлены.

## Чеклист срочного исправления

- Определён текущий тег релиза на production `vX.Y.Z`.
- `hotfix/x.y.z-<short-description>` создан от тега production, а не от `master`.
- Объём срочного исправления минимален и не тянет несвязанные изменения.
- Зафиксирован план возврата изменений в активную `release/x.y` и `master`.
- Проверки для срочного исправления пройдены до выпуска патч-релиза.

## Чеклист выкладки

- Выбран тег релиза `vX.Y.Z` для выкладки.
- Создан и заполнен `docs/releases/vX.Y.Z/release-plan.md`.
- Web и Workers будут выкачены на один и тот же тег.
- В `release-plan.md` зафиксированы миграции, порядок их применения и риск окна несовместимости.
- В `release-plan.md` зафиксирован план исправления без отката.
- Инструкция выкладки использует `git fetch --tags` и checkout по тегу, а не `git pull`.
