# Деплой (Deploy)

**Деплой (deploy)** — публикация изменений на production-сервере с необходимыми действиями: обновление кода, установка зависимостей, миграции, прогрев кэша, перезапуск или перезагрузка сервисов.

## Границы ответственности

- Документ описывает только общие правила и ссылки на актуальные runbook.
- Не выполняй деплой без явного запроса пользователя или ответственного инженера.
- Не выполняй миграции и любые действия на production без явного запроса.
- В рамках PR фиксируй только изменения кода, документов, инструкций и чеклистов.

## Правила

- Production deploy выполняется только по **конкретному release tag** `vX.Y.Z`.
- Деплой из `master` или `release/x.y` branch head запрещён.
- Если релиз включает миграции — деплой должен учитывать порядок действий и риски.
- Production incidents после деплоя закрываются через hotfix или patch release, а не через rollback.

## Рекомендуемый поток

1. Подготовить и стабилизировать active `release/x.y`.
2. Создать и запушить tag `vX.Y.Z`.
3. Выполнить deploy exact `vX.Y.Z` по актуальному runbook.
4. Провести post-check: основные user-flow, логи, метрики.
5. При проблеме остановить rollout и выпустить hotfix или patch release.
